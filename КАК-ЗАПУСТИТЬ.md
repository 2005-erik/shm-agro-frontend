# Сайт ТОО «СХМ Агро»

Одностраничный сайт (SPA) производителя сельхозтехники. Сайт **уже опубликован**:
хостинг — Vercel, база данных и вход в админку — Supabase.

- **Адрес сайта:** `<вставить ссылку с Vercel>`
- **Репозиторий:** https://github.com/2005-erik/shm-agro-frontend
- **Хостинг:** Vercel (статика, деплой из ветки `main`)
- **База:** Supabase — проект `dtmmeyshpfnbxqfctegz`

## Как обновить сайт (основной способ)

Vercel подключён к GitHub. Любой push в `main` автоматически пересобирает сайт —
через 20–40 секунд изменения уже онлайн.

```bash
cd "СХМ Агро/SHM-AGRO-FINAL"
git add -A
git commit -m "что поменял"
git push
```

Проверить статус деплоя: vercel.com → проект → вкладка **Deployments**.
Если деплой красный (Failed) — открыть его, посмотреть лог, откатиться кнопкой
**Promote to Production** на предыдущий зелёный деплой.

## Как посмотреть правки локально (перед push)

Весь сайт — один файл `shm-agro/index.html`. Двойной клик по нему — откроется в браузере.
Локально работает всё, кроме:
- карты Яндекса — нужен интернет;
- AI-чата — нужен бэкенд (см. ниже), без него чат отвечает по встроенной базе.

Сборка не нужна: ни npm, ни node, ни `package.json` — чистая статика.

> В настройках Vercel **Root Directory** должен быть `shm-agro`, Framework Preset — `Other`.
> Build Command и Output Directory — пустые.

## Supabase

Ключи прописаны прямо в `index.html` (строки 14–15) — это **publishable**-ключ,
он и должен быть публичным, прятать его не надо. Защита — через RLS-политики в Supabase.

Таблицы:

| Таблица | Что хранит | Кто пишет |
|---|---|---|
| `models` | техника каталога | админ (пока вручную через Supabase) |
| `news` | новости | админ (пока вручную через Supabase) |
| `requests` | заявки с сайта (КП, звонок, обратная связь) | сайт, автоматически |

Если Supabase недоступен — сайт не падает: подставляются встроенные демо-данные из `index.html`.

## Админка

Открывается ссылкой «Админка» в шапке сайта. Вход — по email и паролю через
Supabase Auth. Пользователей заводить здесь:
Supabase → **Authentication → Users → Add user**.

**Важно, что админка умеет сейчас:**
- ✅ показывает заявки из базы (`requests`) — это боевые данные;
- ⚠️ редактирование техники и новостей **не сохраняется в базу** — правки живут
  только до перезагрузки страницы. Реальные изменения каталога и новостей пока
  вносятся напрямую в Supabase (Table Editor) или правкой `index.html`.

## AI-чат-бот

Кнопка 💬 справа снизу. Два режима:
- **бэкенд запущен** — ответы через AI (Groq);
- **бэкенд выключен** — ответы по локальной базе вопросов, чат не ломается.

Сейчас на живом сайте работает второй режим: в `index.html` адрес бэкенда всё ещё
локальный:

```js
var BACKEND_URL = "http://localhost:3001/api/chat";   // строка ~2034
```

Чтобы включить AI на боевом сайте: задеплоить папку `shm-agro-backend` (Railway,
Render, Vercel Functions — любой), прописать там ключ Groq в переменных окружения
и заменить `BACKEND_URL` на публичный адрес бэкенда.

## Где что править в index.html

- `models: [ ... ]` — техника каталога (название, фото, галерея, характеристики)
- `news: [ ... ]` — новости (заголовок, дата, обложка, текст)
- `certs: [ ... ]` — сертификаты
- Контакты (телефон, адрес, почта) — в футере и блоке «Контакты», искать через Cmd+F
- Карта — блок «Контакты», iframe `yandex.ru/map-widget` (меняется параметр `text=`)
- Ключи Supabase — строки 14–15
- Адрес чат-бэкенда — `BACKEND_URL`, строка ~2034

Изменения в этом файле попадают на сайт только после `git push` (см. выше).

## Что осталось доделать

1. Сохранение правок каталога и новостей из админки в Supabase (сейчас правки локальные).
2. Деплой чат-бэкенда и замена `BACKEND_URL`.
3. Замена демо-данных (техника, новости, сертификаты, адрес) на реальные материалы заказчика.
4. Свой домен вместо адреса `*.vercel.app` (Vercel → Settings → Domains).
